1)자바 객체지향프로그래밍 특징
1. 추상화 - 클래스에서 필요한 공통적인 기능과 속성을 묶어 이름을 부여하는 것을 의미한다.
2. 캡슐화 - 변수와 함수를 같은 목적으로 묶는 것, 데이터를 외부에서 직접 접근막음(private)
3. 상속 - 부모클래스의 속성과 기능을 자식클래스에서 상속받아 사용가능
4. 다형성 - 하나의 변수명,메소드가 상황에 따라 다양한 기능을 하도록 하는것
(부모-자식 상속관계에서 오버라이딩,오버로딩)
2)클래스의 구성요소 - 필드란 무엇인가요?
필드는 데이터를 저장하기 위해 클래스에 선언되는 변수(클래스의 전역변수)

3)클래스의 구성요소 - 메소드란 무엇인가요?
메소드는 필드의 값을 이용한 작업을 수행하도록하는 명령문입니다.

4)클래스의 구성요소 - 생성자란 무엇인가요?
객체를 초기화해주기 위한 메소드로 객체를 생성할때 실행된다.

5)static 필드, static 메소드 : static 키워드의 특징?
메모리를 공유한다. static {}안에서는 static만 사용가능하다
클래스 호출, 객체 생성을 따로 할 필요없이 바로바로 사용가능

6)인스턴스 필드, 인스턴스 메소드 : static 키워드가 없을 때 특징?
메모리에 데이터가 없기때문에 객체를 생성하여 클래스에 접근가능

7)객체의 정의
클래스 설계를 바탕으로 만들어지는 복사본(인스턴스) 입니다.								
new 연산으로 생성됩니다.								
클래스 이름과 동일한 생성자 메소드를 실행하면서 객체가 생성됩니다.

8)클래스의 상속의 주요 특징
부모클래스 지정은 extends 키워드로 합니다.								
1. 생성자는 부모클래스와 관련있다.								
													
2. 부모클래스의 필드와 메소드는 자식클래스가 바로 접근할 수 있다. (super 키워드 생략)								
									
3. 부모클래스의 메소드는 오버라이드(변경,재정의) 할 수 있다. 						
									
4.부모 클래스  필드, 메소드에  protected 키워드를 사용하면 다른 패키지에 있는 자식 클래스는 직접 사용할 수 있다.								
					( - 다른패키지 클래스는 public 키워드 필드,메소드를 사용할 수 있습니다.)				
5. 자식 클래스는 상속 필드 , 상속 메소드 외에 다른 필드와 메소드를 정의할 수 있다.

9)메소드 오버라이딩 정의
부모클래스의 메소드를 자식클래스에서 재정의하여 동작하도록 합니다.(매개변수,리턴타입 같음)

10)메소드 오버로딩 정의
부모클래스의 메소드를 같은 이름으로 다양한 매개변수와 리턴타입의 메소드로 정의하는것
